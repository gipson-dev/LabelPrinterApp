# Security Policy

## Reporting a Vulnerability

Please do not open a public issue for security vulnerabilities.

Instead, report suspected vulnerabilities privately through GitHub's security
advisory tools if available, or contact the repository owner directly.

Include as much detail as possible:

- A clear description of the issue.
- Steps to reproduce the behavior.
- The potential impact.
- Any relevant logs, screenshots, or proof-of-concept details.

## Supported Versions

This project is in early setup and does not have released versions yet. Security
support details will be updated once releases begin.
